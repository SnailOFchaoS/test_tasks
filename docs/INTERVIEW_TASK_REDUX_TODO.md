# Задание на собеседовании: React + Redux — Todo List

**Роль:** React/Next.js разработчик  
**Время:** 60–90 минут  
**Уровень:** middle+  

Задание самодостаточное: выполнять в новом Next.js-проекте или в CodeSandbox. Не связано с другими задачами в репозитории.

---

## Задание

Реализовать **список задач (Todo List)** на React + Redux Toolkit.

### Обязательная часть

1. **State в Redux:**
   - Список задач: каждая задача — `{ id: string, text: string, completed: boolean }`.
   - Фильтр отображения: `'all' | 'active' | 'completed'`.

2. **Действия (actions) и экран:**
   - Добавление задачи (поле ввода + кнопка или Enter).
   - Переключение «выполнено / не выполнено» (клик по задаче или чекбокс).
   - Удаление задачи (кнопка удаления у каждой задачи).
   - Переключатель фильтра: «Все» / «Активные» / «Выполненные» — отображаются только задачи по выбранному фильтру.

3. **Реализация:**
   - Один slice (например, `todosSlice`) с состоянием: массив задач и значение фильтра.
   - Селектор: «отфильтрованные задачи» вынести в отдельную функцию (или в `createSelector`), компонент подписывается на результат селектора, а не фильтрует массив сам в render.
   - Типизация TypeScript: типы для задачи, для state слайса, для RootState.

### Опционально (если остаётся время)

- Кнопка «Очистить выполненные» — удаляет из state все задачи с `completed: true`.
- Сохранение списка в `localStorage` и загрузка при инициализации (через `initialState` или в `configureStore` при создании store, либо в отдельном месте без требования к архитектуре).
- Редактирование текста задачи (двойной клик → инпут → сохранение по Enter или blur).

---

## Что проверяем

- Умение спроектировать state и разбить его на поля (список + фильтр).
- Один slice с несколькими reducers и типом state.
- Селекторы (в т.ч. производный список «отфильтрованные задачи») и подписка компонента на селектор.
- Типизация (интерфейс задачи, state слайса, RootState, при необходимости AppDispatch).
- Базовая вёрстка и UX (достаточно простого списка и кнопок).

---

## Старт «в вакууме»

- Новый проект: `npx create-next-app@latest` и установка `@reduxjs/toolkit`, `react-redux`.
- Либо CodeSandbox: шаблон Next.js + те же пакеты.

Итог: одна страница (или изолированный блок) с полным циклом: добавление, переключение completed, удаление, фильтрация; всё состояние и логика — в Redux, отображение — через селекторы.
